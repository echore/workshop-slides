Part 1 · LLM Foundations

In practice

Two ways to run a model yourself

Same four inputs underneath, either in the cloud or on your machine.

Using a cloud API

when you want to run it from your own code

1

Get an API key. Sign in → Dashboard → API keys.

2

Install the package & make a client. e.g. pip install openai

3

Set the four inputs, then send. system prompt · query · model · temperature

OpenAI provider Claude provider Gemini provider

Repeatable and scriptable across thousands of items.

Processing a dataset costs roughly 1–5 cents per record (a paragraph in, short answer out), so ~$10–50 per 10,000 records, cheaper in batch mode.

Running a model locally

when the data can’t leave your machine

1

Get access to the model. e.g. a Hugging Face token for gated models.

2

Install the library that loads it. e.g. pip install transformers, or a no-code app.

3

Build a pipeline with the same inputs. same four knobs, on your hardware.

4

Generate the answer.

Hugging Face where models live Ollama local runner LM Studio no-code app

Works offline and keeps sensitive data private.

A local model needs about 2 GB of VRAM (your graphics card’s memory) for every billion parameters, less if it’s a compressed version.
