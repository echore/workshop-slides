In practice

Using a cloud API

When you want to run the same task over hundreds of thousands of public records, from your own code.

Has anyone called an API before? What do you think the code actually looks like?

The steps

three things, once

1

Get an API key. Sign in → Dashboard → API keys.

2

Install the package & make a client. e.g. pip install openai

3

Set the four inputs, then send. system prompt · query · model · temperature

OpenAI Claude Gemini

Roughly 1–5 cents per record, so ~$10–50 per 10,000 records, cheaper in batch mode.

Example · a 200,000-report sample from Canada Vigilance

# 1. connect once
client = OpenAI(api_key=MY_KEY)

# 2. the four inputs
r = client.responses.create(
  model="gpt-5",
  instructions="You are a pharmacovigilance analyst.",
  input=f"Group this reaction term into one of: "
        f"cardiac, hepatic, skin, neuro, other. "
        f"Return JSON with term, group, confidence. "
        f"If unclear use other + low.\n{term}",
  temperature=0,
)

# 3. loop the sample (public, 1965–2026)
for term in reaction_terms:
  rows.append(group(term))
What comes back {"term": "hepatocellular injury",
 "group": "hepatic", "confidence": "high"}

A free monthly download from Health Canada, so nothing sensitive leaves your machine. Fixed categories + temperature 0 keep all 200,000 grouped the same way.

Part 1 · LLM Foundations

In practice

Running a model locally

When the data is PHI and can’t leave your machine. Same four inputs, your own hardware.

What would you need in place to run a model on your own machine?

The steps

four things, once

1

Get access to the model. e.g. a Hugging Face token for gated models.

2

Install the library that loads it. e.g. pip install transformers, or a no-code app.

3

Build a pipeline with the same inputs. same four knobs, on your hardware.

4

Generate the answer.

Hugging Face where models live Ollama local runner LM Studio no-code app

About 2 GB of VRAM per billion parameters, less if it’s a compressed version.

Example · 100 EHR notes, on your own machine

# 1–2. load the model onto your machine
pipe = pipeline("text-generation",
  model="meta-llama/Llama-3-8B-Instruct")

# 3. same four inputs as the API
out = pipe(
  f"From this clinical note return smoking "
  f"status, diagnosis, housing.\n{note}",
  max_new_tokens=200, temperature=0)

# 4. loop all 100 notes, offline
for note in ehr_notes:      # 100 rows
  rows.append(extract(note))
What comes back “Pt reports quitting 2 yrs ago, now in shelter…”
→ smoking: former · housing: unstable

All 100 notes stay on your machine. Same four inputs as the API, but nothing crosses the internet, and it finds things the structured fields don’t capture.
