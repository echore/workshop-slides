The principles

Prompt engineering is the discipline that has grown up around a set of best practices for making AI output more reliable, efficient, and accurate.

1

Give direction

Tell it who to be and what style to aim for.

2

Specify format

Say what structure you want the answer in.

3

Show an example

Give one worked case and it matches the pattern.

4

Evaluate quality

Check the output; ask it to rate or fix its own work.

5

Divide the work

Break big tasks into steps, done one at a time.

Part 2 · Prompt Engineering

Give Direction

Tell it who to be and what matters. Same paper, same model. Watch the answer sharpen.

“Summarize this study.” → a long, unfocused wall of text. No clear lead.

Add a persona and a reader → it starts writing for a decision-maker.

Add structure and a length limit → a tight, decision-ready brief.

Change only the role to “peer reviewer” → same paper, now it weighs methods and bias.

Show an Example

Don't describe the format you want. Show one worked case, and the model matches it.

“Extract the key details.” → a loose paragraph. Different fields, different shape every time.

Give one worked example → the study comes back in the exact same fields.

Same prompt, next study → identical structure. Set the pattern once, reuse it across a whole review.

Divide the Work

Break a big task into steps. The more you separate and check them, the more you can trust the result.

One prompt does everything at once. Fluent, but the steps blur and it's hard to see what it skipped.

One prompt, but you spell out the steps. The stages become visible and ordered.

Each step its own prompt. You check and fix each output before it feeds the next. When the model chains its own steps, that's an agent.

Specify Format

Say what shape you want the answer in, and you get output you can use as-is.

“Summarize the key results.” → loose prose. You still have to pull the numbers out by hand.

Ask for a table (Finding, Number, Population) → a clean grid you can paste straight into a review.

Ask for CSV instead → same content, ready to open in Excel. The format is yours to dictate.

Part 2 · Prompt Engineering

Evaluate Quality

Don't take the first answer on faith. Make the model check its own work.

The model gives an answer. It reads well, so it's tempting to trust it as-is.

“Check your answer against the source. Flag anything unsupported.” → it catches its own gaps.

“Rate each claim high / medium / low confidence, with a reason.” → a black box becomes auditable.

Paste the findings into a different model and ask it to rate them → an independent check catches what the first was sure of.
